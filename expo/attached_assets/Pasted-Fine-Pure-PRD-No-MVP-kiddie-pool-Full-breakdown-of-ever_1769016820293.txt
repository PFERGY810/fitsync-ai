1. PRODUCT PURPOSE

A fully autonomous AI hypertrophy system that:
	•	reads physique trends over time
	•	analyzes training data and lifting form
	•	updates macros weekly
	•	rewrites the user’s split based on measurable recovery + volume
	•	acts as a coach, not a logbook

User interacts through photos, videos, and daily logging. App responds by changing their entire program, not just collecting data.

⸻

2. GLOBAL SYSTEM ARCHITECTURE

AI system composed of 5 subsystems:

2.1 PHYSIQUE MODEL

Inputs:
	•	front/side/back photos
	•	lighting/pose normalization
	•	segmentation mask
	•	skeletal landmarks
	•	volumetric estimation model

Outputs:
	•	muscle group score vs prior week
	•	symmetry delta left/right
	•	fat distribution shift
	•	posture + mobility flags
	•	recommendations weight
	•	weak targets priority array

Core Logic:

GrowthScore = (VolumeChangeNormalized * SymmetryCorrectionWeight) – FatGainPenalty


⸻

2.2 FORM MODEL

Inputs:
	•	single-exercise video (bench/squat/row/etc.)
	•	rep segmentation
	•	pose vectors
	•	load metadata
	•	bar path extraction

Outputs:
	•	cue list
	•	warning list (risk flags)
	•	per-rep score
	•	RIR estimation
	•	technical breakdown

Core Logic:

RepQuality = DepthScore + JointIntegrityScore + BarPathStability – VelocityDrop


⸻

2.3 MACRO ENGINE

Inputs:
	•	weight trendline (7-day rolling)
	•	caloric log
	•	training performance
	•	recovery inputs

Outputs:
	•	macros for next 7 days
	•	carb cycling parameters
	•	daily calories

Core Logic:

If WeightTrend >1% loss → +30-50g carbs
If WeightTrend <0.4% loss → -25g carbs


⸻

2.4 SPLIT ENGINE

Inputs:
	•	muscle scores
	•	volume history
	•	soreness score
	•	rep quality
	•	strength curve

Outputs:
	•	new weekly split
	•	sets per muscle
	•	rep targets
	•	RIR targets
	•	tempo cues

Core Logic:

If WeakPointScore[Chest] high → +2 sets/weekly
If FatigueScore high → remove junk sets


⸻

2.5 PROGRESS ENGINE

Inputs:
	•	photo deltas week to week
	•	form reports
	•	load progress
	•	bodyweight trend

Outputs:
	•	combined scoreboard
	•	coach notes
	•	“top 3 fixes” priority instruction

6. AI PIPELINES

6.1 Physique Pipeline
	1.	normalize image orientation
	2.	run human segmentation
	3.	extract skeletal joints
	4.	estimate muscle area by region
	5.	compute deltas week-to-week
	6.	update progress models
	7.	feed weak points into SplitEngine

⸻

6.2 Form Pipeline
	1.	segment reps
	2.	pose detection per frame
	3.	bar path extraction
	4.	angle deviation scoring
	5.	velocity drop per rep
	6.	assign rep quality
	7.	produce cue list
	8.	compute failure point
	9.	store for trend

⸻

6.3 Split Pipeline

Inputs:
	•	WeakPointScore
	•	VolumeHistory
	•	RecoveryScore
	•	ProgressTrend

Process:
	•	adjusts weekly volume per muscle
	•	picks exercise variations
	•	sets RIR targets
	•	assigns unilateral work if asymmetry

⸻

6.4 Macro Pipeline

Inputs:
	•	WeightHistory
	•	CalorieLog
	•	TrainingLoad

Process:
	•	calculates weekly avg
	•	compares to target
	•	calculates ± grams
	•	outputs new macros

8. BEHAVIOR RULES
	•	AI alone controls program changes.
	•	User does not tweak volume manually.
	•	Every 7 days: full re-design of split.
	•	Daily: micro-adaptation from recovery.
	•	Form issues override volume progression.
	•	Asymmetry injects unilateral volume automatically.
